CURL

简介 参数 基础

简介

cURL 官方地址: 主页, 手册,

cURL 是一个利用 URL 语法在命令行下工作的文件传输工具，1997 年首次发行。它支持文件上传和下载，所以是综合传输工具，但按传统，习惯称 cURL 为下载工具。cURL 还包含了用于程序开发的 libcurl。

cURL 支持的通信协议有 FTP、FTPS、HTTP、HTTPS、TFTP、SFTP、Gopher、SCP、Telnet、DICT、FILE、LDAP、LDAPS、IMAP、POP3、SMTP 和 RTSP。

cURL 还支持 SSL 认证、HTTP POST、HTTP PUT、FTP 上传, HTTP form based upload、proxies、HTTP/2、cookies、用户名 + 密码认证(Basic, Plain, Digest, CRAM-MD5, NTLM, Negotiate and Kerberos)、file transfer resume、proxy tunneling。

基础

1.GET 请求(只在 Shell 中展示，二进制可能为乱码)

$ curl https://www.oschina.net/													# html 数据
$ curl -I https://www.oschina.net/action/apiv2/banner?catalog=1					# 只请求响应头
$ curl https://www.oschina.net/action/apiv2/banner?catalog=1					# json 原始数据
$ curl https://www.oschina.net/action/apiv2/banner?catalog=1 | json_pp          # 格式化 json
$ curl https://www.oschina.net/action/apiv2/banner?catalog=1 | json_pp | more   # 分页阅读

$ curl https://www.oschina.net/action/apiv2/banner?catalog=1 -o banner.json     		# 原始数据存储到文件
$ curl https://www.oschina.net/action/apiv2/banner?catalog=1 | json_pp > format.json    # 格式化后存储

2.HTTP 请求

$ curl -d key1=value1&key2=value2 http://example.com				# POST 请求
$ curl -d key1=value1 -d key2=value2 http://example.com				# POST 请求
$ curl --data-urlencode "name=Alan Walker" http://example.com		# 对参数进行 url-encode 编码

$ curl -d '{I Am A JSON FORM}' -H 'Content-Type: application/json' http://example.com # 指定请求类型
#### 当使用 POST 方法提交数据时，对于提交的数据主要有如下四种形式：
#### 	application/x-www-form-urlencoded -- 默认的形式，即 key1=value1&key2=value2 的形式
#### 	multipart/form-data -- 使用表单上传文件时使用这个形式
#### 	application/json -- 提交 JSON 格式的数据
#### 	text/xml -- 提交XML格式的数据

# 当使用 -F 选项时，默认的 Content-Type 就是 multipart/form-data
$ curl -F person=annonymous -F secret=@filename http://example.com/submit.cgi

$ curl http://api.lesson1234.com:8080/lr/ability?action=list -H "channel:lr" -H "version:1" # 修改请求头
$ curl --user-agent "[User Agent]" http://example.com			# 修改 user-agent
$ curl --cookie "CookieName=CookieValue" http://example.com		# 设置 cookie
$ curl -c cookie.jar.txt http://example.com						# 保存 cookie
$ curl -b cookies.txt -c cookie.jar.txt http://example.com		# 设置请求的 cookie,保存返回的 cookie

3.FTP 请求

$ curl -u jiwanger ftp://d.lesson1234.com/						# FTP 用户目录
$ curl -u jiwanger ftp://d.lesson1234.com/108.png -o 108.png    # 输出到指定文件
$ curl -u jiwanger ftp://d.lesson1234.com/108.png -O    		# 使用远程名字保存
